1. An adversary can gain unauthorized access to resources in an Azure subscription  [State: Not Started]  [Priority: High] 

Category:Elevation of Privileges
Description:An adversary can gain unauthorized access to resources in Azure subscription. The adversary can be either a disgruntled internal user, or someone who has stolen the credentials of an Azure subscription.
Justification:<no mitigation provided>
Possible Mitigation(s):Enable fine-grained access management to Azure Subscription using RBAC. Refer: <a href="https://aka.ms/tmtauthz#grained-rbac">https://aka.ms/tmtauthz#grained-rbac</a>
SDL Phase:Design

SDL Phase:Design

3. An adversary may gain unauthorized access to run any action on Azure Data Explorer due to weak authorization rules  [State: Not Started]  [Priority: High] 

Category:Elevation of Privileges
Description:An adversary may gain unauthorized access to run any action on Azure Data Explorer due to weak authorization rules
Justification:<no mitigation provided>
Possible Mitigation(s):Ensure access to run any action on a Kusto resource is restricted by assigning the appropriate Role-Based Access Control (RBAC) roles to only those needing the privileges. Follow the principle of least privilege. Security roles define which security principals (users and applications) can have permissions to operate on a secured resource (such as a database or a table), and what operations are permitted. Refer : 1) <a href="https://aka.ms/tmt-th194 ">https://aka.ms/tmt-th194 </a> 2)<a href="https://aka.ms/tmt-th194a ">https://aka.ms/tmt-th194a </a>
SDL Phase:Implementation

4. Secret information should not be logged in Azure Data Explorer  [State: Not Started]  [Priority: High] 

Category:Information Disclosure
Description:Secret information should not be logged in Azure Data Explorer
Justification:<no mitigation provided>
Possible Mitigation(s):Ensure any secret information like passwords , SAS Tokens , refresh tokens etc are not logged in Azure Data Explorer.
SDL Phase:Implementation

5. Sensitive information might get disclosed while querying Azure Data Explorer  [State: Not Started]  [Priority: Medium] 

Category:Information Disclosure
Description:Sensitive information might get disclosed while querying Azure Data Explorer
Justification:<no mitigation provided>
Possible Mitigation(s):RestrictedViewAccess policy can be enabled on tables in database which contain sensitive information and only principals with "UnrestrictedViewer" role in the database can query that data.Refer : <a href="https://aka.ms/tmt-th196 ">https://aka.ms/tmt-th196 </a>
